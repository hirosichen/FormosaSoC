## 1. SoC 概述

FormosaSoC（福爾摩沙系統單晶片）是一顆基於 RISC-V 開放指令集架構的物聯網（IoT）系統單晶片，使用 LiteX 框架建構。其設計目標為實現台灣自主可控的物聯網晶片，功能對標 ESP32 等主流 IoT SoC。

### 主要規格摘要

| 項目 | 規格 |
|------|------|
| CPU 核心 | VexRiscv (RV32IMC) |
| 系統時脈 | 160 MHz（可依 FPGA 平台調整） |
| 匯流排 | Wishbone B4，32 位元 |
| Boot ROM | 256 KB |
| SRAM | 64 KB |
| GPIO | 32 位元三態 |
| UART | 2 組，115200 baud 預設 |
| SPI Master | 2 組，4 種模式 |
| I2C Master | 2 組，標準/快速模式 |
| PWM | 4 通道（硬體暫存器）/ 8 通道（LiteX） |
| Timer | 4 組 32 位元計時器 |
| Watchdog | 1 組，鎖定保護 |
| ADC | 12 位元 SAR，8 通道 |
| DMA | 4 通道 |
| RTC | 32.768 kHz |
| 中斷控制器 | PLIC，19 個中斷源 |
| Wi-Fi | 802.11a/g OFDM 數位基頻 |
| BLE | Bluetooth 5.0 GFSK 基頻 |

## 3. VexRiscv CPU 核心規格

### 3.1 指令集架構

FormosaSoC 採用 VexRiscv 軟核處理器，為 SpinalHDL 生態系中高度可配置的 RISC-V 處理器 IP。

| 項目 | 規格 |
|------|------|
| ISA | RISC-V RV32IMC |
| 整數指令集 (I) | 基本整數運算、分支、載入/存儲 |
| 乘除法指令集 (M) | 硬體乘法器與除法器 |
| 壓縮指令集 (C) | 16 位元壓縮指令，降低程式碼大小 |
| 特權模式 | Machine Mode (M-mode) |
| 管線級數 | 5 級（Fetch, Decode, Execute, Memory, WriteBack） |
| 指令快取 | 可配置（FPGA 版本預設啟用） |
| 資料快取 | 可配置 |
| 中斷 | 支援 RISC-V 標準中斷（mstatus, mie, mip） |
| CSR | Machine Mode 全套 CSR 暫存器 |
| 除錯介面 | 可選 JTAG 除錯（標準 + debug 變體） |

### 3.2 Machine Mode CSR 暫存器

| CSR 名稱 | 功能 |
|-----------|------|
| `mstatus` | Machine 狀態暫存器（MIE, MPIE, MPP） |
| `mie` | Machine 中斷致能（MSIE, MTIE, MEIE） |
| `mip` | Machine 中斷等待（MSIP, MTIP, MEIP） |
| `mtvec` | Machine 陷阱向量基底位址 |
| `mepc` | Machine 例外程式計數器 |
| `mcause` | Machine 陷阱原因 |
| `mtval` | Machine 陷阱值 |
| `mscratch` | Machine 臨時暫存器 |

### 3.3 中斷控制位元

## 4. Wishbone B4 匯流排架構

### 4.1 匯流排概述

FormosaSoC 使用 Wishbone B4 管線化匯流排作為系統互連。Wishbone 是一個開放標準的晶片內匯流排架構，適用於 SoC 設計。

| 項目 | 規格 |
|------|------|
| 匯流排標準 | Wishbone B4（管線化） |
| 資料寬度 | 32 位元 |
| 位址寬度 | 32 位元 |
| 傳輸模式 | 單一週期、突發傳輸 |
| 位元組選擇 | 4 位元（支援位元組/半字組存取） |

### 4.2 匯流排信號定義

| 信號 | 方向 | 說明 |
|------|------|------|
| `wb_adr` | Master → Slave | 位址匯流排 |
| `wb_dat_w` | Master → Slave | 寫入資料 |
| `wb_dat_r` | Slave → Master | 讀出資料 |
| `wb_we` | Master → Slave | 寫入致能（1=寫入, 0=讀取） |
| `wb_sel` | Master → Slave | 位元組選擇遮罩 |
| `wb_stb` | Master → Slave | 選通信號（表示有效的匯流排週期） |
| `wb_cyc` | Master → Slave | 匯流排週期（表示匯流排被佔用） |
| `wb_ack` | Slave → Master | 應答信號（從端完成傳輸） |

### 4.3 匯流排拓撲

                  各周邊 Wishbone 從端介面
```

## 6. 時脈樹

### 6.1 時脈域定義

FormosaSoC 定義了以下時脈域：

| 時脈域 | 頻率 | 來源 | 用途 |
|--------|------|------|------|
| 系統主時脈 (SYS) | 160 MHz | PLL 輸出 | CPU 核心、Wishbone 匯流排 |
| AHB 匯流排時脈 | 80 MHz | SYS / 2 | 高頻周邊、DMA、Wi-Fi 基頻 |
| APB 匯流排時脈 | 40 MHz | SYS / 4 | 低速周邊（UART, SPI, I2C, PWM, Timer） |
| 外部晶振 (XTAL) | 40 MHz | 外部晶振 | PLL 參考時脈 |
| RTC 時脈 | 32.768 kHz | 外部晶振 | RTC 計時、低功耗計時 |

### 6.2 時脈樹結構

```

                    ┌────┴─────┐
                    │   PLL    │
                    │          │
                    │  倍頻: x4│
                    │  分頻: /1│
                    └────┬─────┘
                         │
                   160 MHz (SYS_CLK)
                         │
            ┌────────────┼────────────┐
            │            │            │

       │  /1     │  │  /2    │  │  /4    │
       │ 160 MHz │  │ 80 MHz │  │ 40 MHz │

### 6.3 FPGA 平台時脈

不同 FPGA 平台的實際時脈配置：

|-----------|-----------|-------------------|
| Tang Nano 20K | 27 MHz | 48 MHz |

## 7. 重設架構

### 7.1 重設源

FormosaSoC 支援多種重設來源：

| 重設源 | 描述 | 暫存器 |
|--------|------|--------|
| 上電重設 (POR) | 電源上電時自動觸發系統重設 | 硬體 |
| 外部重設 | 外部重設腳位（低電位有效） | 硬體 |
| 軟體重設 | CPU 透過暫存器觸發全域重設 | `RSTCTRL_SW_RST` |
| 看門狗重設 | 看門狗計時器逾時觸發重設 | `WDT_CTRL` |

### 7.2 重設控制器

```
               ┌────────────┐
  上電重設 ───→│            │
  外部重設 ───→│  重設      │──→ 全域重設信號
  軟體重設 ───→│  OR 閘     │    (同步至 SYS_CLK)
  看門狗重設──→│  +同步器   │
               └────────────┘
                     │
                     ▼
            ┌────────────────┐
            │ 重設分配矩陣    │
            ├────────────────┤
            │ CPU 核心重設    │
            │ 匯流排重設      │
            │ 周邊重設        │
            │ 無線模組重設    │
            └────────────────┘
```

### 7.3 軟體重設

軟體重設透過寫入 `RSTCTRL_SW_RST` 暫存器觸發：

```c
/* 觸發全域軟體重設 */
RSTCTRL_SW_RST = RSTCTRL_SW_RST_KEY;  /* 0x5A5A0001 */
```

必須寫入正確的魔術數字 `0x5A5A0001` 才能觸發重設，防止誤操作。

### 7.4 周邊獨立重設

各周邊可透過 `RSTCTRL_PERIPH_RST` 暫存器獨立觸發軟體重設，寫入 1 觸發重設，硬體自動清除：

```
RSTCTRL_PERIPH_RST 暫存器 (偏移 0x04)：
寫入對應位元為 1 即觸發該周邊重設
```

### 7.5 重設原因查詢

透過 `RSTCTRL_RST_STATUS` 暫存器（唯讀）可查詢最近一次重設的原因。

## 8. 電源域

### 8.1 電源模式

FormosaSoC 支援以下電源管理模式：

| 模式 | 描述 | 功耗 | 喚醒源 |
|------|------|------|--------|
| 正常運行 (Active) | 所有模組運行 | 最高 | - |
| 睡眠模式 (Sleep) | CPU 暫停，周邊持續運行 | 中 | 中斷 |
| 深度睡眠 (Deep Sleep) | 僅 RTC 與喚醒邏輯運行 | 低 | RTC 鬧鐘、GPIO |
| 休眠模式 (Hibernate) | 僅保留 SRAM 內容 | 最低 | 外部腳位 |

### 8.2 電源控制暫存器

```
SYSCTRL_PWR_CTRL 暫存器 (偏移 0x0C)：
┌──────┬───────────┬──────────────┬─────────┐
│ 位元 │     2     │      1       │    0    │
├──────┼───────────┼──────────────┼─────────┤
│ 名稱 │ HIBERNATE │ DEEP_SLEEP   │ SLEEP   │
│ 功能 │ 進入休眠  │ 進入深度睡眠  │進入睡眠 │
└──────┴───────────┴──────────────┴─────────┘
```

### 8.3 電壓域

| 電壓域 | 電壓 | 供電範圍 |
|--------|------|---------|
| 核心電壓 | 1.2V | CPU 核心、SRAM、邏輯 |
| I/O 電壓 | 3.3V | GPIO、UART、SPI、I2C |
| 類比電壓 | 3.3V | ADC、PLL |
| RTC 電壓 | 1.2V | RTC 計時器 |

## 9. 中斷架構

### 9.1 PLIC 概述

FormosaSoC 採用 RISC-V 標準的平台級中斷控制器（Platform-Level Interrupt Controller, PLIC），負責管理所有外部中斷源。

| 項目 | 規格 |
|------|------|
| 中斷源數量 | 19 個（最多支援 64 個） |
| 優先權等級 | 可配置，每個中斷源獨立設定 |
| 致能控制 | 每個中斷源獨立致能/停用 |
| 領取/完成機制 | 硬體自動仲裁，軟體領取並完成 |

### 9.2 中斷源分配

| IRQ 編號 | 中斷源 | 優先權群組 |
|----------|--------|-----------|
| 0 | 保留（無中斷） | - |
| 1 | GPIO | 通用周邊 |
| 2 | UART0 | 通訊 |
| 3 | UART1 | 通訊 |
| 4 | SPI0 | 通訊 |
| 5 | SPI1 | 通訊 |
| 6 | I2C0 | 通訊 |
| 7 | I2C1 | 通訊 |
| 8 | PWM | 通用周邊 |
| 9 | Timer 0 | 系統核心 |
| 10 | Timer 1 | 系統核心 |
| 11 | Timer 2 | 系統核心 |
| 12 | Timer 3 | 系統核心 |
| 13 | WDT | 系統核心 |
| 14 | ADC | 通用周邊 |
| 15 | DMA | 系統核心 |
| 16 | RTC | 系統核心 |
| 17 | Wi-Fi | 無線通訊 |
| 18 | BLE | 無線通訊 |

### 9.3 中斷處理流程

```
  外部中斷源         PLIC              CPU (mstatus.MIE=1)
  ┌────────┐    ┌──────────┐        ┌──────────────┐
  │ GPIO   ├───→│ 優先權   │        │              │
  │ UART   ├───→│ 仲裁     ├──────→│  外部中斷    │
  │ SPI    ├───→│          │ meip   │  mip.MEIP=1  │
  │  ...   ├───→│ 致能遮罩 │        │              │
  └────────┘    └──────────┘        └──────┬───────┘
                                           │
                                    ┌──────┴───────┐
                                    │ 跳至 mtvec   │
                                    │ 中斷處理常式 │
                                    └──────┬───────┘
                                           │
                                    ┌──────┴───────┐
                                    │ 1. 讀 PLIC   │
                                    │    CLAIM      │
                                    │ 2. 執行 ISR  │
                                    │ 3. 寫 PLIC   │
                                    │    COMPLETE   │
                                    └──────────────┘
```

### 9.4 中斷處理程式碼範例

```c
#include "formosa_soc.h"

/* 初始化 PLIC */
void plic_init(void)
{
    /* 設定所有中斷優先權為 1 */
    for (int i = 1; i < IRQ_MAX; i++) {
        PLIC_PRIORITY(i) = 1;
    }

    /* 設定優先權閾值為 0（接受所有優先權 > 0 的中斷） */
    PLIC_THRESHOLD = 0;
}

/* 致能指定中斷 */
void plic_enable_irq(uint32_t irq)
{
    PLIC_ENABLE |= (1UL << irq);
}

/* 中斷處理常式 */
void isr_handler(void)
{
    /* 領取中斷 */
    uint32_t irq = PLIC_CLAIM;

    /* 根據中斷源分派處理 */
    switch (irq) {
        case IRQ_GPIO:    gpio_irq_handler();  break;
        case IRQ_UART0:   uart0_irq_handler(); break;
        case IRQ_TIMER0:  timer0_irq_handler();break;
        /* ... 其他中斷源 ... */
        default: break;
    }

    /* 完成中斷 */
    PLIC_CLAIM = irq;
}
```
